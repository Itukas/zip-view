## 新增需求

### 需求:按文件类型路由到内置查看器

系统必须根据条目的文件类型，将其优先路由到对应的内置查看器（图片→图册查看器、文本→文本预览等）。

#### 场景:打开图片走内置图册
- **当** 用户点击一个图片条目
- **那么** 系统使用内置图册查看器打开它

#### 场景:打开文本走内置预览
- **当** 用户点击一个文本条目
- **那么** 系统使用内置文本预览打开它

#### 场景:未知类型提供打开方式选择
- **当** 用户点击一个无内置查看器支持的条目
- **那么** 系统提供"用其他应用打开"的选项

### 需求:转交外部应用打开

系统必须支持将压缩包内条目转交给外部应用打开（通过系统 Intent），且无论条目是否有内置查看器，用户都必须能选择"用其他应用打开"。

#### 场景:用其他应用打开条目
- **当** 用户对某条目选择"用其他应用打开"
- **那么** 系统将该条目内容提供给系统并以 Intent 方式转交可处理的外部应用

#### 场景:无可用外部应用
- **当** 用户选择"用其他应用打开"但系统中没有可处理该类型的应用
- **那么** 系统提示没有可用的应用
