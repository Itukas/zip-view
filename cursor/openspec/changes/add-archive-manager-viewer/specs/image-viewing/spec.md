## 新增需求

### 需求:免解压查看包内图片

系统必须能够对压缩包内的单张图片直接开流解码并显示，禁止为查看单张图片而将整个压缩包解压到磁盘（对支持随机访问的格式）。

#### 场景:直接查看 zip 内图片
- **当** 用户在 zip/cbz 中点击一张图片
- **那么** 系统对该图片条目开流解码并显示，不在磁盘落地整包解压

### 需求:图册式翻页浏览

系统必须把压缩包内的图片集合作为图册，支持在图片间翻页（前一张/后一张），并预加载相邻图片以保证翻页流畅。

#### 场景:翻到下一张图片
- **当** 用户在查看器中向后翻页
- **那么** 系统显示下一张图片，并已提前预加载其后续图片

#### 场景:预加载窗口受限
- **当** 系统预加载相邻图片
- **那么** 预加载范围必须限制在当前图片前后各 1~2 张以内，避免内存膨胀

### 需求:图片缩放查看

系统必须支持对图片进行缩放与平移查看。

#### 场景:双指缩放图片
- **当** 用户在查看器中对图片做双指缩放手势
- **那么** 系统相应放大或缩小该图片并支持平移

### 需求:大图内存防护

系统必须对图片解码进行降采样，禁止因加载超大图片而导致内存溢出。

#### 场景:加载超大图片
- **当** 用户查看一张分辨率极大的图片
- **那么** 系统按显示需要降采样解码，避免 OOM
