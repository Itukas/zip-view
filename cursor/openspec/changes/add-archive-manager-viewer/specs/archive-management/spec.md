## 新增需求

### 需求:接入压缩包来源

系统必须支持用户通过系统文件选择器（SAF）选择压缩包，以及通过"分享到本应用"的 Intent 接收压缩包，并将可访问的压缩包加入应用内列表。系统禁止在首版强制申请全盘存储权限。

#### 场景:通过文件选择器添加压缩包
- **当** 用户在应用内点击"添加"并通过系统文件选择器选中一个压缩包
- **那么** 系统持久化该文件的可访问授权并将其加入压缩包列表

#### 场景:从外部分享接收压缩包
- **当** 用户在其他应用中将一个压缩包"分享"到本应用
- **那么** 系统接收该压缩包并将其展示在列表中（或直接打开浏览）

#### 场景:授权失效时引导重新选择
- **当** 列表中某压缩包的访问授权已失效（如授权被撤销）
- **那么** 系统提示该项不可访问并引导用户重新选择文件

### 需求:展示压缩包列表

系统必须以列表形式展示已接入的压缩包，每项至少显示名称、格式与大小。

#### 场景:查看压缩包列表
- **当** 用户打开应用主界面
- **那么** 系统展示所有已接入压缩包的名称、格式和大小

### 需求:压缩包基础管理操作

系统必须支持对列表中的压缩包执行打开、重命名、删除以及查看信息等基础操作。

#### 场景:删除压缩包条目
- **当** 用户对某压缩包选择"删除"并确认
- **那么** 系统将其从列表移除（按用户选择决定是否删除源文件）

#### 场景:查看压缩包信息
- **当** 用户对某压缩包选择"信息"
- **那么** 系统展示其名称、格式、大小、条目数量等元信息
