## 为什么

很多人把漫画、扫描图集、表情包等打包成 zip/rar 等压缩包存在手机里。现有文件管理器要么不能直接看包内图片（必须先整包解压，产生大量碎文件、占空间又麻烦），要么看图体验很差。我们要做一个 Android app，把"压缩包管理"和"免解压直接看图"合二为一：既能像文件管理器一样管理压缩包，又能把一个压缩包当成"一本可以直接翻的图册"来看，无需解压。

## 变更内容

- 新增一个 Android app（APK），首版交付以下核心能力：
  - **管理压缩包**：接入本地存储/系统文件/分享进来的压缩包，列表展示并支持基础管理（重命名、删除、查看信息等）。
  - **免解压浏览包内结构**：打开压缩包时只读目录表，秒速列出包内文件/文件夹，支持进入子目录、查看嵌套压缩包，全程不在磁盘落地解压文件。
  - **免解压图册查看器**：直接对包内单张图片开流解码显示，支持翻页、缩放、预加载下一张，把压缩包当图册翻。
  - **选择打开方式**：内置查看器优先（图片→图册、文本→预览等），按文件类型路由；同时支持"用其他应用打开"，把条目转交系统/外部 app（Intent）。
- **格式策略**：架构按"全格式"设计（zip / rar / 7z / tar 等通过统一 Provider 抽象接入），但实现分批落地。首版优先保证 `zip`/`cbz` 的满级"免解压秒开"体验；`rar`/`7z`/`tar` 等按 Provider 逐步补齐，对天生不支持随机抽取的格式（如 7z solid、tar.gz）采用"后台解到缓存"的降级方案。
- **UI 原则**：简易明了，不堆砌功能，突出"列表 → 浏览 → 看图"的主路径。

## 功能 (Capabilities)

### 新增功能
- `archive-management`: 压缩包的接入与管理——文件来源（本地/SAF/分享 Intent）、压缩包列表展示、基础文件操作（打开、重命名、删除、信息）。
- `archive-browsing`: 免解压浏览压缩包内部结构——通过统一 Provider 抽象列出条目、进入子目录、处理多种压缩格式与嵌套压缩包，并定义各格式的支持级别与降级策略。
- `image-viewing`: 免解压图册查看器——对包内图片直接开流解码、翻页、缩放、预加载与内存/OOM 防护。
- `open-with`: 打开方式路由——按文件类型选择内置查看器，或将条目转交外部应用（Intent）。

## 影响

- **新增代码**：全新的 Android 应用工程（Kotlin），含 UI 层、ArchiveProvider 抽象层与各格式适配、文件来源/存储访问层、图片解码与缓存。
- **依赖**：图片加载库（Glide/Coil）、压缩格式库（zip 用 JDK 原生；rar 用 junrar；7z 用 sevenzipjbinding 或 commons-compress；tar 用 commons-compress）。
- **平台**：Android 存储权限与 SAF（Storage Access Framework）、分享 Intent 接入、`minSdk`/`targetSdk` 选择。
- **非目标（首版）**：压缩/打包新压缩包、压缩包加密创建、云盘同步——首版聚焦"管理 + 浏览 + 看图"。
